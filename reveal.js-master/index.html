Same Origin Policy

Hintergrund

Problem: Skriptsprachen im Browser haben direkten Zugriff auf die gesamte Kommunkation zwischen Browser und Web-Servern

Idee: Informationen aus einem Skript-Kontext dürfen nicht von einem Skript aus einem anderen Kontext zugreifbar oder manipulierbar sein

Vergleich der origin

Client: http://ma.dhbw.de/dir/seite.html

Server Erfolg Grund
http://ma.dhbw.de/dir/andereseite.html Ja
http://ma.dhbw.de/ordner/seite.html Ja
http://ma.dhbw.de:81/dir/seite.html Nein Port
https://ma.dhbw.de/dir/seite.html Nein Protokoll
http://st.dhbw.de/dir/seite.html Nein Host

Herkunft = Protokoll + Domain + Port

Problematik

  • Vor Angriffsmethoden nicht ausreichend geschützt
  • Grenzen des SOP oftmals nicht erwünscht

Cross-Origin Resource Sharing

Kompromiss zugunsten größerer Flexibilität im Internet, unter Berücksichtung möglichst hoher Sicherheitsmaßnahmen.

Fuktionsweise

Einschränkungen können vom Server für bestimmte Clients aufgehoben werden.

Dies geschieht durch spezifischen HTTP-Header.
